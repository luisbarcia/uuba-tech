### 2. API Backend (api.uuba.tech)

**Sprint 1 — Fundação:**
- Scaffold FastAPI + Docker Compose + PostgreSQL 16
- Config via pydantic-settings, database async (SQLAlchemy 2.0)
- Autenticação por API Key (timing-safe com hmac)
- Error handling RFC 9457 (Problem Details)
- Request ID middleware (X-Request-Id em toda resposta)
- Structured logging JSON
- Health check endpoint

**Sprint 2 — Core CRUD:**
- 3 models: Cliente, Fatura, Cobrança (SQLAlchemy 2.0 async)
- 14 endpoints REST:
  - Clientes: criar, listar, buscar, atualizar, métricas financeiras
  - Faturas: criar, listar, buscar, atualizar (status→pago auto-preenche pago_em)
  - Cobranças: criar, listar, histórico por fatura, pausar, retomar
- Paginação padronizada (envelope list com total, has_more, offset)
- Alembic migrations configurado
- Filtros: por status, cliente, telefone, período
