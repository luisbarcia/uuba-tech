1 Contexto

A Evolution API emula o WhatsApp Web (não oficial). Empresas médias e grandes não aceitam:

  • Sem contrato com Meta
  • Risco de ban a qualquer momento
  • Sem SLA
  • Não passa por compliance/jurídico

Solução: Dual Gateway

Adicionar suporte à WhatsApp Business API (Meta Cloud API) como canal oficial. Manter Evolution como opção para operações menores ou desenvolvimento.

┌─────────────────────┐ │ n8n Workflow │ │ (lógica de negócio) │ └──────────┬──────────┘ │ ┌──────────▼──────────┐ │ Gateway Adapter │ │ (sub-workflow n8n) │ └───┬─────────────┬───┘ │ │ ┌─────────▼───┐ ┌─────▼─────────┐ │ Evolution │ │ Meta Cloud API │ │ (dev/PME) │ │ (enterprise) │ └─────────────┘ └────────────────┘

Templates de mensagem (HSM)

Para cobrança proativa, é obrigatório usar templates pré-aprovados pela Meta:

Nome: cobranca_lembrete Categoria: Utility Idioma: pt_BR Corpo: Olá {{1}}, identificamos uma pendência no valor de R$ {{2}} com vencimento em {{3}}. Podemos ajudar a resolver?
Importante

Respostas dentro da janela de 24h (cliente iniciou) não precisam de template. O bot responde livremente.

4 Fases de implementação

Fase 1 — Setup Meta 1-2 dias + 3-7 dias verificação

  • Criar Meta Business Account com CNPJ
  • Criar app, adicionar WhatsApp
  • Verificar o negócio
  • Obter número, gerar token permanente
  • Configurar webhook → n8n

Fase 2 — Normalizer de entrada 1 dia

Sub-workflow que converte webhooks de ambos os formatos para formato interno padrão:

{ "source": "meta" | "evolution", "phone": "5521999999999", "name": "Nome do contato", "message": "Texto da mensagem", "messageId": "wamid.xxx", "timestamp": 1679000000, "type": "text" | "image" | "audio" | "document" }

Fase 3 — Gateway Router de saída 1 dia

Substitui o "WhatsApp Enviar" atual. Verifica config e envia pelo gateway correto.

Fase 4 — Templates de cobrança 1 dia

  • Criar 3-5 templates na Meta, aguardar aprovação
  • Implementar envio de template no Gateway Router
  • Manter envio livre para respostas na janela de 24h

Fase 5 — Chatwoot 0.5 dia

Configurar inbox oficial WhatsApp Business no Chatwoot (integração nativa v3.x).

Fase 6 — Testes 1-2 dias

  • Recebimento e resposta via Meta Cloud API
  • Cobrança proativa com template
  • Escalação para Chatwoot
  • Teste com número real de cliente

Fase 7 — Go-live 0.5 dia

Documentação + go-live gradual (1 cliente → 10 → todos).

Webhook Meta — verificação

A Meta faz um GET com hub.verify_token + hub.challenge. Retornar o challenge em plain text.

Validação de assinatura

Todo POST da Meta vem com header X-Hub-Signature-256. Validar com HMAC-SHA256 usando o App Secret.

Node nativo do n8n

O n8n tem node "WhatsApp Business Cloud" que suporta envio de texto, template, mídia e mensagens interativas. Pode substituir o HTTP Request node no Gateway Router.

Variáveis de ambiente novas

META_PHONE_NUMBER_ID= META_ACCESS_TOKEN= META_APP_SECRET= META_VERIFY_TOKEN= META_WABA_ID= WHATSAPP_GATEWAY=evolution|meta

8 Chatwoot

Opção A — API Inbox

n8n encaminha para Chatwoot quando escala (mesmo modelo atual)

Opção B — Nativo ✓

Chatwoot tem integração nativa com WhatsApp Cloud API desde v3.x. Configurar direto no painel.

Recomendação: Opção B para canal oficial + Opção A para Evolution.

12 Checklist de go-live

Antes de ativar

  • Meta Business Account verificada
  • Número de telefone aprovado e ativo
  • Access Token permanente gerado (System User)
  • Webhook configurado e respondendo (GET + POST)
  • Assinatura HMAC validada em todos os webhooks
  • Templates de cobrança aprovados (mínimo 3)
  • Normalizer testado com payloads reais
  • Gateway Router testado para envio
  • Chatwoot recebendo conversas do canal oficial
  • Escalação bot → humano funcionando
  • Monitoramento de custos configurado

Primeiro dia

  • Enviar 5-10 cobranças pelo canal oficial
  • Monitorar taxa de entrega e leitura
  • Verificar bot responde dentro da janela de 24h
  • Confirmar Evolution funcionando em paralelo

Primeira semana

  • Expandir para todos os clientes enterprise
  • Monitorar custos reais vs estimativa
  • Ajustar templates se taxa de resposta baixa
  • Documentar issues encontrados
Resumo para holdings/clientes

"A UÚBA suporta dois canais de WhatsApp: a API oficial da Meta (WhatsApp Business Platform) para clientes que exigem compliance e SLA, e um canal alternativo para operações menores. A arquitetura é agnóstica ao gateway — toda a lógica de cobrança, IA e integração funciona independentemente do canal."
